# دليل عمل البوت — كيف يعمل البوت؟

## نظرة عامة

هذا البوت هو **نظام فلترة وأرشفة طبية ذكي** يعمل داخل تيليجرام. هو مزيج من:
- **UserBot**: يتحكم في حسابات تيليجرام الشخصية عبر بروتوكول MTProto
- **Control Bot**: واجهة تحكم تتحدث معها من هاتفك لإعطاء الأوامر

---

## ملفات المشروع

| الملف | الوظيفة |
|-------|---------|
| `main.py` | نقطة الدخول — يشغل البوت ويدير لوحة التحكم |
| `config.py` | الإعدادات والكلمات المفتاحية وأسماء القنوات |
| `classifier.py` | محرك التصنيف الهرمي بالكلمات المفتاحية |
| `harvester.py` | حصاد الروابط من مجموعات المصادر |
| `sorter.py` | فرز وتصنيف الروابط وإرسال التقارير |
| `account_manager.py` | إدارة الحسابات المتعددة والجلسات |
| `channel_setup.py` | إنشاء قنوات الأرشيف الخمس تلقائياً |
| `database.py` | إدارة البيانات والذاكرة |
| `.env.example` | قالب الإعدادات السرية |

## مراحل العمل

### المرحلة 1: الإعداد
1. تحصل على `API_ID` و `API_HASH` من [my.telegram.org](https://my.telegram.org)
2. تحصل على `BOT_TOKEN` من [@BotFather](https://t.me/botfather)
3. تعرف `OWNER_ID` الخاص بك من [@userinfobot](https://t.me/userinfobot)
4. تنسخ `.env.example` إلى `.env` وتملأ البيانات
5. تثبت المكتبات: `pip install -r requirements.txt`
6. تشغل البوت: `python main.py`

### المرحلة 2: ربط الحسابات
- تضغط زر **"ربط حساب جديد"**
- يطلب منك البوت رقم الهاتف
- يرسل تيليجرام كود تحقق لحسابك
- تدخل الكود والبوت يحفظ الجلسة في مجلد `sessions/`
- يمكن إضافة حسابات متعددة لتوزيع الحمل

### المرحلة 3: إنشاء قنوات الأرشيف
- تضغط **"إنشاء قنوات الأرشيف"**
- يُنشئ البوت تلقائياً القنوات التالية:
  - 📋 أرشيف - طب بشري عام
  - 🔪 أرشيف - جراحة
  - 🦷 أرشيف - أسنان
  - 💊 أرشيف - صيدلة
  - 🔬 أرشيف - مختبرات
  - 👶 أرشيف - طب أطفال
  - 🩺 أرشيف - تمريض
  - 📚 أرشيف - كتب ومراجع
  - 💬 أرشيف - استفسارات
  - 🎓 أرشيف - ابتعاث ومنح
  - ⚠️ أرشيف - روابط تالفة وخاصة
- يحفظ IDs القنوات في `bot_memory.json`

### المرحلة 4: إضافة المصادر
- تضغط **"إضافة مصادر"**
- ترسل روابط مجموعات الروابط (كل رابط في سطر)
- يدعم الروابط العادية `t.me/group` والروابط الخاصة `t.me/+xyz`

### المرحلة 5: حصاد الروابط
- تضغط **"حصاد الروابط"**
- البوت يمر على كل رسالة في كل مجموعة مصدر
- يستخرج **كل** الروابط من كل رسالة (حتى لو كانت الرسالة تحتوي 100 رابط)
- يحذف المكرر ويحفظ النتائج في `raw_links.json`
- يُحدّثك بالتقدم كل 200 رسالة

### المرحلة 6: الفرز والتصنيف
- تضغط **"بدء الفرز الشامل"**
- يبدأ البوت بمعالجة كل رابط من القائمة الخام

#### ما يفعله البوت لكل رابط:
1. **التحقق من التكرار** — إذا سبق فحصه، يتخطاه فوراً
2. **كشف نوع الرابط:**
   - `addlist/` → يفك ضغطه ويستخرج القنوات الداخلية
   - `t.me/+` → رابط دعوة خاص → ينقله لقناة "تالفة وخاصة"
   - رابط عادي → يفحصه عميقاً
3. **الفحص العميق (بدون انضمام):**
   - يجلب اسم القناة/المجموعة
   - يجلب الوصف (Bio)
   - يجلب عدد الأعضاء
   - يتحقق هل الحساب منضم أم لا
   - يحدد النوع (قناة، مجموعة، بوت)
4. **التصنيف الهرمي:**
   - يدمج (الاسم + الوصف + المعرف) في نص واحد
   - يبحث بالكلمات المفتاحية بترتيب هرمي
   - يضع الرابط في التصنيف المناسب
5. **الإرسال:**
   - يرسل تقريراً منظماً للقناة المناسبة
   - يتضمن: الاسم، الرابط، النوع، التصنيف، الأعضاء، حالة الانضمام، اسم الحساب الفاحص

## نظام الحماية من الحظر (Anti-Ban)

| الإجراء | التفاصيل |
|---------|---------|
| **التأخير العشوائي** | 3-7 ثواني بين كل عملية فحص |
| **تبديل الحسابات** | كل 100 عملية ينتقل للحساب التالي |
| **الاستراحة الوقائية** | كل 500 عملية يأخذ استراحة 5 دقائق |
| **منع التكرار** | لا يفحص الرابط مرتين عبر أي حساب |
| **نقطة التوقف** | يحفظ مكانه في `bot_memory.json` لاستئناف العمل |

## التقارير المُرسلة للقنوات

كل رابط يُرسل تقريراً على الشكل:
```
📌 الاسم: [اسم القناة/المجموعة]
🔗 الرابط: https://t.me/...
🏷 النوع: 📢 قناة / 👥 مجموعة / 🤖 بوت
🧬 التصنيف: [التخصص الطبي]
👥 الأعضاء: [العدد]
✅ منضم؟: نعم / لا
👤 فحص بواسطة: [اسم الحساب]
📝 الوصف: [أول 200 حرف من الوصف]
```

## ميزة معالجة المجلدات (Addlist)

عند اكتشاف رابط `t.me/addlist/xxx`:
- يفك البوت ضغط المجلد
- يستخرج جميع القنوات الداخلية
- يضيفها للقائمة الحالية ويفحصها كروابط فردية
