# ما الذي لم يتم تضمينه في الكود؟

هذا الملف يوثق الأشياء التي تمت الإشارة إليها في المحادثة ولكن تطلب قراراً أو معلومات إضافية منك قبل التنفيذ، أو هي ميزات اختيارية تحتاج إضافتها لاحقاً.

---

## 1. ميزات تحتاج بيانات منك لإضافتها

### 🔑 بيانات الاعتماد (API Keys)
- `API_ID` و `API_HASH` — يجب الحصول عليها من [my.telegram.org](https://my.telegram.org)
- `BOT_TOKEN` — يجب الحصول عليه من [@BotFather](https://t.me/BotFather)
- `OWNER_ID` — معرفك الشخصي، احصل عليه من [@userinfobot](https://t.me/userinfobot)
- **هذه البيانات لا يمكن تعيينها في الكود لأسباب أمنية** — أنت تضعها في ملف `.env`

---

## 2. ميزات مذكورة في المحادثة لكنها بحاجة إلى عمل إضافي

### 🤖 الذكاء الاصطناعي للتصنيف (AI Classification)
- **ما ذُكر:** استخدام `huggingface_hub` للتصنيف بالذكاء الاصطناعي بدلاً من الكلمات المفتاحية فقط
- **لماذا لم يُضف:** يتطلب نموذجاً عربياً مخصصاً وتثبيت مكتبات ثقيلة، وقد تحدث المحادثة عنه كخيار مستقبلي
- **للإضافة:** قم بتوسيع دالة `classify_specialty()` في `classifier.py` لاستخدام نموذج `aubmindlab/bert-base-arabertv02-twitter` مثلاً

### 📅 جدولة المهام (Task Scheduling)
- **ما ذُكر:** استخدام `aiocron` لجدولة الحصاد والفرز تلقائياً بدون تدخل يدوي
- **لماذا لم يُضف:** تحتاج لتحديد وقت/جدول معين — هذا قرار شخصي
- **للإضافة:** `pip install aiocron` وإضافة `@aiocron.crontab("0 */6 * * *")` قبل دالة الحصاد

### 📊 تصدير Excel احتياطي
- **ما ذُكر:** تصدير النتائج إلى ملف Excel (`pandas` + `openpyxl`) كنسخة احتياطية
- **لماذا لم يُضف:** الكود يركز على إرسال النتائج للقنوات (الطريقة الرئيسية). Excel كان يُذكر كبديل
- **للإضافة:** `pip install pandas openpyxl` وإضافة دالة في `sorter.py` لحفظ `results` كـ DataFrame

### 🔢 إحصائيات مفصلة لكل تصنيف
- **ما ذُكر:** إحصائيات تُظهر كم رابط في كل تخصص طبي
- **ما هو موجود:** إحصائيات إجمالية (مرتبة، تالفة، مكررة)
- **للإضافة:** في `database.py`، أضف عداداً لكل تصنيف في `db["stats"]`

## 3. ميزات مذكورة وتم تنفيذها بشكل جزئي

### ⚠️ روابط الدعوة (t.me/+)
- **ما هو موجود:** يكتشفها ويرسلها لقناة "تالفة وخاصة"
- **ما لم يُضف بالكامل:** محاولة فحصها مرة واحدة من "حساب التضحية" كما ذُكر في المحادثة
- **السبب:** هذا يتطلب تحديد أي حساب هو "حساب التضحية" — قرار منك

### 🗂️ تصنيف الروابط للقنوات الخمس الأولى (channels/groups/bots/addlists/broken)
- **ما هو موجود:** يصنف حسب التخصص الطبي (جراحة، أسنان...) ويتضمن معلومة النوع في التقرير
- **ما ذُكر أيضاً:** إنشاء 5 قنوات أساسية منفصلة (channels, groups, bots, addlists, broken) **بالإضافة** للتصنيف الطبي
- **الحل المقترح:** يمكن دمج الاثنين بإنشاء قنوات للتخصصات الطبية (المُنفذ حالياً) مع ذكر النوع في التقرير (مُنفذ)
- **للإضافة الكاملة:** أضف `channel_type_channels` في `config.py` لإنشاء 5 قنوات إضافية بالتوازي مع التصنيف الطبي

## 4. تحسينات مستقبلية مقترحة في المحادثة

| التحسين | الوصف |
|---------|-------|
| **منع التكرار بقاعدة بيانات** | استبدال `global_seen.txt` بـ SQLite أو PostgreSQL للأداء مع ملايين الروابط |
| **معالجة أسرع** | إضافة `asyncio.gather()` لفحص عدة روابط بالتوازي بحذر |
| **إشعارات الانتهاء** | إرسال تقرير نهائي PDF أو Excel عند اكتمال الجلسة |
| **واجهة ويب** | لوحة تحكم ويب (باستخدام Flask أو FastAPI) لعرض الإحصائيات |
| **البحث داخل الأرشيف** | أمر `/search` يبحث في الروابط المُرشفة |

---

## 5. ما تحتاج فعله قبل تشغيل البوت

1. **إنشاء ملف `.env`:**
   ```
   API_ID=ضع_رقمك
   API_HASH=ضع_هاشك
   BOT_TOKEN=ضع_التوكن
   OWNER_ID=ضع_معرفك
   ```

2. **تثبيت المكتبات:**
   ```bash
   cd bot
   pip install -r requirements.txt
   ```

3. **تشغيل البوت:**
   ```bash
   python main.py
   ```

4. **في تيليجرام:** ابحث عن بوتك وأرسل `/start`
